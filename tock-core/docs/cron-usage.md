# Cron 使用说明（当前实现）

本文档说明 `com.clmcat.tock.cron.CronCalculator` 当前支持的 Cron 语法、组合方式、边界行为和示例。

## 1. 基本用法

```java
CronCalculator calculator = new CronCalculator(ZoneId.of("Asia/Shanghai"));

Optional<Long> next = calculator.getNextExecutionTime("0 0 9 * * ?", System.currentTimeMillis());
if (next.isPresent()) {
    ZonedDateTime time = Instant.ofEpochMilli(next.get()).atZone(ZoneId.of("Asia/Shanghai"));
}
```

### 方法说明

| 方法 | 说明 |
|---|---|
| `getNextExecutionTime(String cronExpression, long baseTimestamp)` | 计算 **严格晚于** `baseTimestamp` 的下一次执行时间。 |
| `isValidCron(String cronExpression)` | 仅校验表达式语法和字段范围是否合法。 |

## 2. 表达式结构

当前支持 **6 位** 和 **7 位** Cron。

| 位置 | 字段 | 范围 | 说明 |
|---|---|---:|---|
| 1 | 秒 | 0-59 | 必填 |
| 2 | 分 | 0-59 | 必填 |
| 3 | 时 | 0-23 | 必填 |
| 4 | 日 | 1-31 | 支持 `*`、`?`、`L`、`W` |
| 5 | 月 | 1-12 | 必填 |
| 6 | 周 | 1-7 | `1=周日，2=周一...7=周六` |
| 7 | 年 | 1970-9999 | 可选 |

### 重要约定

- `getNextExecutionTime` 返回的是 **下一次**，不是“当前时刻本身”。
- 如果 7 位表达式不写年，系统会按未来年份搜索。
- 当前实现最多向未来搜索 **400 年**。
- `isValidCron` 只代表**语法合法**，不代表在所有年份都有可执行点。

## 3. 通用语法能力

以下语法可用于秒 / 分 / 时 / 月 / 年，以及部分日 / 周字段：

| 语法 | 示例 | 说明 |
|---|---|---|
| `*` | `*` | 当前字段取所有合法值 |
| `n` | `9` | 固定在某一个值 |
| `n,m,k` | `1,5,9` | 枚举多个值 |
| `n-k` | `8-18` | 范围 |
| `*/n` | `*/15` | 从最小值开始按步长递增 |
| `n/n` | `5/10` | 从固定值开始按步长递增 |
| `n-k/n` | `8-18/2` | 在范围内按步长递增 |

### 说明

- 列表、范围、步长可以混用在同一字段中，例如：`5-15/5,20`
- 步长必须大于 0。
- 字段值越界会直接抛出异常。

## 4. 各字段支持情况

### 4.1 秒 / 分 / 时 / 月 / 年

这 5 个字段都支持：

- `*`
- `n`
- `n,m,k`
- `n-k`
- `*/n`
- `n/n`
- `n-k/n`

### 4.2 日字段

日字段支持以下特殊值：

| 语法 | 说明 | 示例 |
|---|---|---|
| `?` | 不指定 | `0 0 12 ? * 2` |
| `L` | 当月最后一天 | `0 0 0 L * ?` |
| `W` | 最近工作日 | `0 0 9 15W * ?` |

#### `W` 规则

- `1W` 表示“最接近 1 号的工作日”
- 如果目标日是周六，通常向前取周五
- 如果目标日是周日，通常向后取周一
- 如果请求日超出当月天数，会先截断到月末，再计算最近工作日

#### `L` + 周字段

当前实现允许日字段写 `L`，同时周字段写具体约束，用来表示：

- “**该月最后一个满足周字段条件的日期**”

例如：

- `0 0 0 L * 3`：每月最后一个周二

### 4.3 周字段

周字段支持以下特殊值：

| 语法 | 说明 | 示例 |
|---|---|---|
| `?` | 不指定 | `0 0 12 * * ?` |
| `n` | 固定星期几 | `0 0 12 * * 2` |
| `n,m,k` | 多个星期几 | `0 0 12 * * 2,4,6` |
| `n-k` | 星期范围 | `0 0 12 * * 2-6` |
| `n/n` | 星期步长 | `0 0 12 * * 2/2` |
| `nL` | 当月最后一个星期几 | `0 0 18 ? * 6L` |
| `n#k` | 当月第 k 个星期几 | `0 0 9 ? * 2#1` |

#### 周字段编号

| 值 | 含义 |
|---|---|
| 1 | 周日 |
| 2 | 周一 |
| 3 | 周二 |
| 4 | 周三 |
| 5 | 周四 |
| 6 | 周五 |
| 7 | 周六 |

## 5. 日字段和周字段的组合规则

当前实现的规则是：

1. 日字段和周字段 **不能同时为 `?`**
2. 日字段和周字段 **不能同时都是“具体约束”**，除非日字段是 `L`
3. 如果日字段是 `L`，可以和周字段一起使用

### 可接受

- `0 0 12 * * ?`
- `0 0 12 ? * 2`
- `0 0 0 L * 3`

### 不可接受

- `0 0 12 ? * ?`
- `0 0 12 15 * 2`

## 6. 常见示例

| Cron | 含义 |
|---|---|
| `0 0 9 * * ?` | 每天 09:00:00 |
| `0 */15 * * * ?` | 每 15 分钟一次 |
| `0 0 8-18/2 * * ?` | 每天 8:00~18:00 之间每 2 小时一次 |
| `10,20,30 0 8-10 * 1,6 ?` | 1 月或 6 月的 8~10 点，在第 10/20/30 秒触发 |
| `0 0 0 L * ?` | 每月最后一天 00:00:00 |
| `0 0 18 ? * 6L` | 每月最后一个周五 18:00:00 |
| `0 0 0 L * 3` | 每月最后一个周二 00:00:00 |
| `0 0 9 15W * ?` | 每月最接近 15 号的工作日 09:00:00 |
| `0 0 0 31W 4 ?` | 4 月 31W 会先截断到月末，再取最近工作日 |
| `0 0 9 ? * 2#1` | 每月第 1 个周一 09:00:00 |
| `0 0 0 29 2 ?` | 仅在闰年触发 2 月 29 日 |
| `0 0 0 29 2 ? 2028` | 仅在 2028 年的闰日触发 |
| `0 30 2 8 3 ? 2026` | DST 春季跳时场景，可能顺延到 03:30 |
| `0 30 1 1 11 ? 2026` | DST 秋季回拨场景，可能存在两个 01:30 |

## 7. DST / 时区行为

当前实现基于构造时传入的 `ZoneId` 计算。

### Spring Forward（春季跳时）

如果目标本地时间不存在，例如 `America/New_York` 在夏令时开始时的 `02:30`，系统会顺延到跳变后的首个有效本地时间，例如 `03:30`。

### Fall Back（秋季回拨）

如果目标本地时间在回拨窗口出现两次，例如 `01:30`，系统会按基准时间选择下一次仍然可触发的那个时刻：

- 基准时间在第一次 `01:30` 前：返回第一次
- 基准时间在第一次 `01:30` 后：返回第二次

## 8. 校验与错误处理

- 语法错误、字段越界、非法组合：抛 `IllegalArgumentException`
- 合法但无未来执行点：返回 `Optional.empty()`
- 例如非闰年显式指定 `2 月 29 日`，会返回空结果而不是异常

## 9. 当前不支持的写法

下面这些写法当前不支持：

- `LW`
- `W` 出现在周字段
- `C`
- 周字段使用 `0` 表示周日
- 非法范围，例如 `18-8`
- 步长为 0，例如 `*/0`

## 10. 推荐写法

如果你只需要常见场景，优先使用这些模式：

- 每天固定时间：`0 0 9 * * ?`
- 每隔 N 分钟：`0 */N * * * ?`
- 每月最后一天：`0 0 0 L * ?`
- 每月最后一个周几：`0 0 18 ? * 6L`
- 每月第 N 个周几：`0 0 9 ? * 2#1`
- 最近工作日：`0 0 9 15W * ?`
