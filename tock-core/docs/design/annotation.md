# Annotation 模块设计

`annotation` 目录目前只有 `TockComponent`。

## 设计说明

- 用于标记 `Tock` 需要装配和管理的组件字段
- 作用是让 `Tock` 的装配链路更清晰，方便生命周期排序和上下文注入
- 这是框架内部标识，不是业务注解

## 依赖链路

- `Tock` 和 `Config` 中的关键字段会被该标记识别
- `LifecycleSupport` / 装配逻辑会据此识别需要管理的组件

## 为什么这样设计

- **减少显式装配样板**：让框架自己识别可管理组件
- **保持内部约束**：这是框架元信息，不影响业务层
